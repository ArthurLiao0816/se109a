# *git* Learning
---

## 版本控制系統
* 版本控制の意思：紀錄檔案的變化(eg.結構、內容...等)，之後要用可以用
* 版本控制の種類：
    * 本地端版本控制 Local VCS：
        * 在自己的電腦上用 `Ctrl + C` 跟 `Ctrl + V` 對檔案進行管理
        * *缺點*：不能協同開發
    * 集中化的版本控制系統 CVS：
        * 用一個伺服器統一管理所有版本的檔案，客戶端再從伺服器拿檔案出來用
        * *缺點*：伺服器掛了就完蛋了
    * 分散式版本控制系統 DVCSs：
        * 用戶端在跟伺服器要資料的時候，會把整個倉儲做鏡像
        * *缺點*：保密性差

## *git* の特性
* 記錄檔案快照，而不是差異
* 沒網路?沒關係，一樣可以做事，有網路再上傳就好
* 用 `checksum` 阻止非正當修改檔案
* 通常都只增加資料(大一用到現在，基本都在傳資料...
* 將檔案分成四種狀態：
    * `committed`：已經上傳惹 ~
    * `modified`：被改了 ~
    * `staged`：等著要被上傳中...
    * `untracked`：新增的檔案 ~
